# Production MLOps Architecture

This document explains how the complete local production-grade MLOps platform fits together.

## Architecture Flow

```text
Developer
  ↓
Git Feature Branch
  ↓
Pull Request
  ↓
CI Pipeline
  ↓
ML Pipeline
  ↓
MLflow Experiment Tracking
  ↓
Hyperparameter Tuning
  ↓
MLflow Model Registry
  ↓
Model Promotion
  ↓
FastAPI Serving
  ↓
Docker Compose / Kubernetes
  ↓
Prometheus Metrics
  ↓
Grafana Dashboards
  ↓
OpenTelemetry Traces
  ↓
Jaeger Trace UI
  ↓
JSON Logs
  ↓
Loki + Promtail
  ↓
Evidently Drift Reports
  ↓
Retraining Trigger
```

## Core Components

| Component | Responsibility |
|---|---|
| Git | Version control |
| GitHub Actions | CI/CD and production workflow |
| Pandas / NumPy | Data processing |
| Scikit-Learn | Logistic Regression model |
| Pytest | Automated tests |
| MLflow Tracking | Experiment tracking |
| MLflow Registry | Model versioning and promotion |
| Hyperopt | Hyperparameter tuning |
| FastAPI | Model serving |
| Docker | Local packaging |
| Docker Compose | Local multi-service runtime |
| Prometheus | Metrics collection |
| Grafana | Metrics and log dashboards |
| OpenTelemetry | Distributed traces |
| Jaeger | Trace visualization |
| Python JSON Logging | Structured logs |
| Loki | Log storage |
| Promtail | Log collection |
| Locust | Load testing |
| Evidently | Data and prediction drift reports |
| Kubernetes | Production-style orchestration |

## Local Docker Compose Runtime

```text
docker-compose.yml
```

Runs:

```text
FastAPI API
MLflow Tracking Server
Prometheus
Grafana
OpenTelemetry Collector
Jaeger
Loki
Promtail
```

Primary service URLs:

```text
API:        http://127.0.0.1:8000
MLflow:     http://127.0.0.1:5000
Prometheus: http://127.0.0.1:9090
Grafana:    http://127.0.0.1:3000
Jaeger:     http://127.0.0.1:16686
Loki:       http://127.0.0.1:3100
```

## Kubernetes Runtime

```text
k8s/
```

Deploys:

```text
Namespace
API Deployment
API Service
API ConfigMap
API PVC
API Ingress
Prometheus
Grafana
Jaeger
OpenTelemetry Collector
Loki
Promtail
```

Deployment command:

```bash
./scripts/deploy_k8s.sh
```

Smoke test command:

```bash
./scripts/smoke_test_k8s_api.sh
```

Cleanup command:

```bash
./scripts/delete_k8s.sh
```

## Production Flow

The production flow is orchestrated by:

```text
src/mlops_lr/production_flow.py
```

And wrapped by:

```text
scripts/run_production_flow.sh
```

It runs:

```text
Quality Checks
ML Pipeline
Tuning Pipeline
Drift Pipeline
Retraining Pipeline
Release Manifest
Docker Build
```

## Release Manifest

The release manifest is generated by:

```text
src/mlops_lr/release_manifest.py
```

Output:

```text
reports/release_manifest.json
```

It records:

```text
Git Branch
Git Commit
Docker Image
Docker Tag
MLflow Tracking URI
MLflow Experiment
Registered Model
Serving Stage
Service URLs
```

## Monitoring

The API exposes Prometheus metrics at:

```text
/metrics
```

Important metrics:

```text
http_requests_total
http_request_duration_seconds
http_errors_total
model_drift_detected
```

## Tracing

The API sends OpenTelemetry traces to:

```text
OpenTelemetry Collector
  ↓
Jaeger
```

Jaeger service name:

```text
mlops-logistic-regression-api
```

## Logging

The API emits JSON logs.

Docker Compose log flow:

```text
API Container Logs
  ↓
Promtail
  ↓
Loki
  ↓
Grafana
```

Kubernetes log flow:

```text
Pod Logs
  ↓
Promtail DaemonSet
  ↓
Loki
  ↓
Grafana
```

## Drift Monitoring

Drift monitoring is handled by:

```text
src/mlops_lr/drift_pipeline.py
```

It creates:

```text
Input Statistics
Prediction Statistics
Evidently Data Drift Report
Evidently Prediction Drift Report
Drift Alert
Retraining Trigger
```

Main outputs:

```text
reports/data_drift.html
reports/prediction_drift.html
reports/drift_alert.json
reports/retraining_trigger.json
```

## Retraining

Retraining is handled by:

```text
src/mlops_lr/retraining_pipeline.py
```

The pipeline reads:

```text
reports/retraining_trigger.json
```

If retraining is required, it runs the ML pipeline again.
